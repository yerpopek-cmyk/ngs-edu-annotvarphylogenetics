## вљЎ Quickstart (5 minutes)

### 1 вЂ” Clone and generate test data

```bash
git clone https://github.com/your-username/ngs-education-hub.git
cd ngs-education-hub

# Generate tiny synthetic datasets (no downloads needed)
python3 scripts/generate_test_data.py --outdir test_data/
```

### 2 вЂ” Run the annotation pipeline

```bash
conda env create -f environments/annotation.yml
conda activate annotation

# Edit config: set ASSEMBLY to your FASTA, or use the test genome
nano pipelines/annotation/config.sh

bash pipelines/annotation/run_annotation.sh
# First run: downloads B. subtilis genome (~4 MB) and Swiss-Prot subset (~40 MB) automatically
```

### 3 вЂ” Run the variant calling pipeline

```bash
conda env create -f environments/variants.yml
conda activate variants

# Edit config with your paths (or use generated test data)
nano pipelines/variants/config.sh

bash pipelines/variants/run_all.sh
```

### 4 вЂ” Run the phylogenetics pipeline

```bash
conda env create -f environments/phylogenetics_env.yml
conda activate phylogenetics_env

bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i test_data/phylogenetics/test_16S.fasta \
    -t DNA
```

## рџ”¬ Genome Annotation

**Question answered:** Given a raw genome assembly (FASTA), where are the genes and what do they do?

### Three-stage workflow

```
Assembly (FASTA)
       в”‚
       в–ј Stage 1: Structural annotation (Prokka)
  Gene coordinates  в”Ђв”Ђв†’  .gff, .faa, .ffn
       в”‚
       в–ј Stage 2: Homology annotation (DIAMOND)
  Function from Swiss-Prot hits  в”Ђв”Ђв†’  blastp.tsv
       в”‚
       в–ј Stage 3: Domain annotation (HMMER)
  Conserved domains from Pfam  в”Ђв”Ђв†’  domtblout.txt
```

### Running

```bash
conda activate annotation

# Default run (downloads B. subtilis 168 genome automatically):
bash pipelines/annotation/run_annotation.sh

# Your own genome (offline):
# Edit ASSEMBLY in pipelines/annotation/config.sh, then:
bash pipelines/annotation/run_annotation.sh --offline

# Limit hmmscan to first 200 proteins for a quick test:
bash pipelines/annotation/run_annotation.sh --max-proteins 200
```

### Outputs

```
outputs/run_YYYYMMDD_HHMMSS/
в”њв”Ђв”Ђ prokka/
в”‚   в”њв”Ђв”Ђ GENOME.gff   в†ђ gene coordinates (GFF3 format)
в”‚   в”њв”Ђв”Ђ GENOME.faa   в†ђ protein sequences (key input for DIAMOND + HMMER)
в”‚   в”њв”Ђв”Ђ GENOME.ffn   в†ђ nucleotide gene sequences
в”‚   в””в”Ђв”Ђ GENOME.txt   в†ђ annotation statistics
в”њв”Ђв”Ђ diamond/
в”‚   в””в”Ђв”Ђ blastp.tsv   в†ђ homology hits (tab-separated)
в””в”Ђв”Ђ hmmer/
    в””в”Ђв”Ђ domtblout.txt в†ђ domain hits (one line per domain)
```

## рџ§¬ Variant Calling

**Question answered:** Given sequencing reads and a reference genome, which positions in our sample differ from the reference вЂ” and which differences are real?

### Five-step workflow

```
FASTQ reads
     в”‚
     в–ј  01_align.sh        BWA mem + samtools markdup
   BAM (sorted, deduped)
     в”‚
     в–ј  02_call.sh         FreeBayes (haplotype-based Bayesian caller)
   raw VCF
     в”‚
     в–ј  03_filter.sh       bcftools norm + soft-filter (LowQual, LowAB)
   filtered VCF
     в”‚
     в–ј  04_annotate.sh     Ensembl VEP (consequence, gnomAD AF, ClinVar)
   annotated VCF
     в”‚
     в–ј  05_report.py       Priority scoring в†’ TSV + Markdown report
   report.md + .tsv
```

### Running

```bash
conda activate variants

# 1. Configure paths (required)
nano pipelines/variants/config.sh
# Set: REF_FASTA, READS_R1, READS_R2, SAMPLE_ID

# 2. Run everything
bash pipelines/variants/run_all.sh

# Re-run from a specific step (e.g., after changing filter thresholds):
bash pipelines/variants/run_all.sh --from 03 --run-dir pipelines/variants/outputs/run_YYYYMMDD_HHMMSS

# Use an existing BAM instead of FASTQ:
# Set BAM_INPUT="/path/to/sample.bam" in config.sh
```

### Outputs

```
outputs/run_YYYYMMDD_HHMMSS/
в”њв”Ђв”Ђ 1_bams/
в”‚   в”њв”Ђв”Ђ SAMPLE.bam          в†ђ sorted + deduplicated BAM
в”‚   в””в”Ђв”Ђ SAMPLE.flagstat     в†ђ alignment QC statistics
в”њв”Ђв”Ђ 2_vcf_raw/
в”‚   в””в”Ђв”Ђ SAMPLE.raw.vcf.gz   в†ђ unfiltered FreeBayes calls
в”њв”Ђв”Ђ 3_vcf_filtered/
в”‚   в”њв”Ђв”Ђ SAMPLE.filtered.vcf.gz    в†ђ normalized + soft-filtered
в”‚   в”њв”Ђв”Ђ SAMPLE.annotated.vcf.gz   в†ђ VEP annotations added
в”‚   в””в”Ђв”Ђ SAMPLE.stats.txt          в†ђ bcftools stats (Ti/Tv, counts)
в””в”Ђв”Ђ 4_reports/
    в”њв”Ђв”Ђ SAMPLE.prioritized.tsv    в†ђ all PASS variants with scores
```

## рџЊі Phylogenetics

**Question answered:** Given a set of sequences, how are they related evolutionarily, and how confident are we in those relationships?

### Seven-step workflow

```
Unaligned FASTA
       в”‚
       в–ј Step 1  MAFFT            Multiple sequence alignment
       в–ј Step 2  ClipKit          Trim uninformative columns
       в–ј Step 3  FastTree         Quick approximate ML tree (exploratory)
       в–ј Step 4  IQ-TREE          Full ML tree + model selection + bootstrap
       в–ј Step 5  gotree           Midpoint rerooting
       в–ј Step 6  BioPython        Convert to PHYLIP (for PhyloBayes)
       в–ј Step 7  BioPython        Generate BEAST2 alignment XML
```

### Running

```bash
conda activate phylogenetics_env

# With the generated 16S test dataset:
bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i test_data/phylogenetics/test_16S.fasta \
    -t DNA

# Your own sequences:
bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i my_proteins.fasta \
    -t AA

# Skip IQ-TREE for a fast exploratory run:
bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i sequences.fasta \
    --skip-iqtree

# Skip steps you've already completed:
bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i sequences.fasta \
    --skip-mafft --skip-clipkit
```

### Outputs

```
outputs/test_16S_results/
в”њв”Ђв”Ђ test_16S_aligned.fasta       в†ђ MAFFT alignment
в”њв”Ђв”Ђ test_16S_trimmed.fasta       в†ђ ClipKit-trimmed alignment
в”њв”Ђв”Ђ test_16S_fasttree.tree       в†ђ FastTree quick tree (Newick)
в”њв”Ђв”Ђ test_16S_iqtree.treefile     в†ђ IQ-TREE ML tree (Newick)
в”њв”Ђв”Ђ test_16S_iqtree.log          в†ђ best-fit substitution model
в”њв”Ђв”Ђ test_16S_rooted.tree         в†ђ midpoint-rooted tree
в”њв”Ђв”Ђ test_16S_trimmed.phy         в†ђ PHYLIP format (for PhyloBayes)
в”њв”Ђв”Ђ test_16S_phylip_name_map.tsv в†ђ PHYLIP ID в†” original name mapping
в”њв”Ђв”Ђ test_16S_beast_alignment.xml в†ђ BEAST2 XML alignment draft
в””в”Ђв”Ђ pipeline.log                 в†ђ complete run log
```
