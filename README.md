> **This is a fully educational repository designed for students, educators, and bioinformatics teams.** It combines three production-grade NGS analysis pipelines—genome annotation, variant calling, and phylogenetics—with detailed theory, step-by-step explanations, and synthetic test datasets for hands-on learning.

## 🗺️ What Is This?

This repository teaches NGS bioinformatics by doing. Every script contains inline explanations of **why** each command exists, what the key parameters mean, and what the output represents. The theory docs connect the code to the underlying biology and statistics.

### Three Pipelines, One Repo

| Pipeline | What It Solves | Key Tools |
|----------|---------------|-----------|
| [🔬 Genome Annotation](#-genome-annotation) | "What genes are in this assembly?" | Prokka · DIAMOND · HMMER |
| [🧬 Variant Calling](#-variant-calling) | "How does this sample differ from the reference?" | BWA · FreeBayes · VEP |
| [🌳 Phylogenetics](#-phylogenetics) | "How are these sequences related evolutionarily?" | MAFFT · ClipKit · IQ-TREE |

## ⚡ Quickstart (5 minutes)

### 1 — Clone and generate test data

```bash
git clone https://github.com/your-username/ngs-education-hub.git
cd ngs-education-hub

# Generate tiny synthetic datasets (no downloads needed)
python3 scripts/generate_test_data.py --outdir test_data/
```

### 2 — Run the annotation pipeline

```bash
conda env create -f environments/annotation.yml
conda activate annotation

# Edit config: set ASSEMBLY to your FASTA, or use the test genome
nano pipelines/annotation/config.sh

bash pipelines/annotation/run_annotation.sh
# First run: downloads B. subtilis genome (~4 MB) and Swiss-Prot subset (~40 MB) automatically
```

### 3 — Run the variant calling pipeline

```bash
conda env create -f environments/variants.yml
conda activate variants

# Edit config with your paths (or use generated test data)
nano pipelines/variants/config.sh

bash pipelines/variants/run_all.sh
```

### 4 — Run the phylogenetics pipeline

```bash
conda env create -f environments/phylogenetics_env.yml
conda activate phylogenetics_env

bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i test_data/phylogenetics/test_16S.fasta \
    -t DNA
```

## 🔬 Genome Annotation

**Question answered:** Given a raw genome assembly (FASTA), where are the genes and what do they do?

### Three-stage workflow

```
Assembly (FASTA)
       │
       ▼ Stage 1: Structural annotation (Prokka)
  Gene coordinates  ──→  .gff, .faa, .ffn
       │
       ▼ Stage 2: Homology annotation (DIAMOND)
  Function from Swiss-Prot hits  ──→  blastp.tsv
       │
       ▼ Stage 3: Domain annotation (HMMER)
  Conserved domains from Pfam  ──→  domtblout.txt
```

### Running

```bash
conda activate annotation

# Default run (downloads B. subtilis 168 genome automatically):
bash pipelines/annotation/run_annotation.sh

# Your own genome (offline):
# Edit ASSEMBLY in pipelines/annotation/config.sh, then:
bash pipelines/annotation/run_annotation.sh --offline

# Limit hmmscan to first 200 proteins for a quick test:
bash pipelines/annotation/run_annotation.sh --max-proteins 200
```

### Outputs

```
outputs/run_YYYYMMDD_HHMMSS/
├── prokka/
│   ├── GENOME.gff   ← gene coordinates (GFF3 format)
│   ├── GENOME.faa   ← protein sequences (key input for DIAMOND + HMMER)
│   ├── GENOME.ffn   ← nucleotide gene sequences
│   └── GENOME.txt   ← annotation statistics
├── diamond/
│   └── blastp.tsv   ← homology hits (tab-separated)
└── hmmer/
    └── domtblout.txt ← domain hits (one line per domain)
```

📖 Theory: [`docs/01_genome_annotation.md`](docs/01_genome_annotation.md)

## 🧬 Variant Calling

**Question answered:** Given sequencing reads and a reference genome, which positions in our sample differ from the reference — and which differences are real?

### Five-step workflow

```
FASTQ reads
     │
     ▼  01_align.sh        BWA mem + samtools markdup
   BAM (sorted, deduped)
     │
     ▼  02_call.sh         FreeBayes (haplotype-based Bayesian caller)
   raw VCF
     │
     ▼  03_filter.sh       bcftools norm + soft-filter (LowQual, LowAB)
   filtered VCF
     │
     ▼  04_annotate.sh     Ensembl VEP (consequence, gnomAD AF, ClinVar)
   annotated VCF
     │
     ▼  05_report.py       Priority scoring → TSV + Markdown report
   report.md + .tsv
```

### Running

```bash
conda activate variants

# 1. Configure paths (required)
nano pipelines/variants/config.sh
# Set: REF_FASTA, READS_R1, READS_R2, SAMPLE_ID

# 2. Run everything
bash pipelines/variants/run_all.sh

# Re-run from a specific step (e.g., after changing filter thresholds):
bash pipelines/variants/run_all.sh --from 03 --run-dir pipelines/variants/outputs/run_YYYYMMDD_HHMMSS

# Use an existing BAM instead of FASTQ:
# Set BAM_INPUT="/path/to/sample.bam" in config.sh
```

### Outputs

```
outputs/run_YYYYMMDD_HHMMSS/
├── 1_bams/
│   ├── SAMPLE.bam          ← sorted + deduplicated BAM
│   └── SAMPLE.flagstat     ← alignment QC statistics
├── 2_vcf_raw/
│   └── SAMPLE.raw.vcf.gz   ← unfiltered FreeBayes calls
├── 3_vcf_filtered/
│   ├── SAMPLE.filtered.vcf.gz    ← normalized + soft-filtered
│   ├── SAMPLE.annotated.vcf.gz   ← VEP annotations added
│   └── SAMPLE.stats.txt          ← bcftools stats (Ti/Tv, counts)
└── 4_reports/
    ├── SAMPLE.prioritized.tsv    ← all PASS variants with scores
    └── SAMPLE.report.md          ← top-N variants + scoring explanation
```

> ⚠️ **VEP cache:** Step 04 requires a local Ensembl VEP cache (~15 GB for GRCh38).
> If the cache is absent, the script downloads it automatically the first time.
> Set `VEP_CACHE_DIR` in `config.sh` to choose where it lives.

## 🌳 Phylogenetics

**Question answered:** Given a set of sequences, how are they related evolutionarily, and how confident are we in those relationships?

### Seven-step workflow

```
Unaligned FASTA
       │
       ▼ Step 1  MAFFT            Multiple sequence alignment
       ▼ Step 2  ClipKit          Trim uninformative columns
       ▼ Step 3  FastTree         Quick approximate ML tree (exploratory)
       ▼ Step 4  IQ-TREE          Full ML tree + model selection + bootstrap
       ▼ Step 5  gotree           Midpoint rerooting
       ▼ Step 6  BioPython        Convert to PHYLIP (for PhyloBayes)
       ▼ Step 7  BioPython        Generate BEAST2 alignment XML
```

### Running

```bash
conda activate phylogenetics_env

# With the generated 16S test dataset:
bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i test_data/phylogenetics/test_16S.fasta \
    -t DNA

# Your own sequences:
bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i my_proteins.fasta \
    -t AA

# Skip IQ-TREE for a fast exploratory run:
bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i sequences.fasta \
    --skip-iqtree

# Skip steps you've already completed:
bash pipelines/phylogenetics/run_phylogenetics.sh \
    -i sequences.fasta \
    --skip-mafft --skip-clipkit
```

### Outputs

```
outputs/test_16S_results/
├── test_16S_aligned.fasta       ← MAFFT alignment
├── test_16S_trimmed.fasta       ← ClipKit-trimmed alignment
├── test_16S_fasttree.tree       ← FastTree quick tree (Newick)
├── test_16S_iqtree.treefile     ← IQ-TREE ML tree (Newick)
├── test_16S_iqtree.log          ← best-fit substitution model
├── test_16S_rooted.tree         ← midpoint-rooted tree
├── test_16S_trimmed.phy         ← PHYLIP format (for PhyloBayes)
├── test_16S_phylip_name_map.tsv ← PHYLIP ID ↔ original name mapping
├── test_16S_beast_alignment.xml ← BEAST2 XML alignment draft
└── pipeline.log                 ← complete run log
```

## 🧪 Generating Test Data

All three pipelines can be tested with tiny synthetic datasets generated locally — no downloads needed, runs in < 30 seconds.

```bash
python3 scripts/generate_test_data.py --outdir test_data/ --seed 42
```

**What gets created:**

| File | Description | Pipeline |
|------|-------------|----------|
| `test_data/phylogenetics/test_16S.fasta` | 20 synthetic 16S sequences (~500 bp each) | Phylogenetics |
| `test_data/annotation/test_genome.fasta` | 1 synthetic bacterial contig (10 kb) | Annotation |
| `test_data/variants/test_ref.fasta` | Synthetic reference (50 kb) | Variants |
| `test_data/variants/test_R1/R2.fastq.gz` | 5 000 paired-end reads × 150 bp (~30× coverage) | Variants |

```
1. docs/glossary.md              ← Learn the vocabulary
```
