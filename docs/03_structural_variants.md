# 3. Структурные варианты (SV) и изменения копийности (CNV/CNA)

## 3.1. Особенности SV/CNV

Структурные варианты (SV) и изменения копийности (CNV/CNA) имеют фундаментальные отличия от малых вариантов (SNV/indel):

| Характеристика | Малые варианты | Структурные варианты |
|----------------|----------------|---------------------|
| **Размер** | 1–50 bp | >50 bp до нескольких Mb |
| **Локализация** | Точечная позиция | Распределено по региону |
| **Детекция** | Прямое выравнивание | Косвенные сигналы |
| **Сложность** | Относительно простая | Высокая (повторы, дупликации) |

### Основные сигналы в BAM

| Сигнал | Описание | Что обнаруживает |
|--------|----------|------------------|
| **Split reads** | Одно чтение «разрывается» на два места | Точные точки разрыва |
| **Discordant pairs** | Ориентация/дистанция парных чтений не соответствует ожидаемой | Инверсии, транслокации, крупные инсерции/делеции |
| **Coverage shift** | Изменение глубины покрытия | CNV (амплификации, делеции) |

### Главные враги при детекции SV ⚠️

1. 🔴 **Повторы** — множественные копии последовательностей
2. 🔴 **Сегментные дупликации** — большие дуплицированные регионы
3. 🔴 **Низкая mappability** — чтения картируются неоднозначно
4. 🔴 **GC-bias** — систематические искажения покрытия

## 3.2. VCF для SV/CNV

Формат VCF для структурных вариантов расширен специальными полями:

| Поле | Описание |
|------|----------|
| **SVTYPE** | Тип структурного варианта (DEL, DUP, INV, INS, BND, CNV) |
| **END** | Конечная позиция варианта |
| **SVLEN** | Длина варианта (может быть отрицательной для делеций) |
| **CIPOS** | Интервал неопределённости для позиции начала |
| **CIEND** | Интервал неопределённости для позиции конца |
| **IMPRECISE** | Флаг неточного вызова |
| **SUPPORT** | Количество поддерживающих чтений |

### Важное замечание

Один и тот же SV может быть описан по-разному разными инструментами. **Бенчмаркинг строится на overlap (перекрытии), а не точном совпадении координат.**

**Пример записи SV в VCF:**

```
chr1  10000  .  N  <DEL>  30  PASS  SVTYPE=DEL;END=15000;SVLEN=-5000;CIPOS=-50,50;CIEND=-50,50  GT:DV:RV  0/1:15:12
```

## 3.3. Типы структурных вариантов

| Тип | Обозначение | Описание | Размер |
|-----|-------------|----------|--------|
| **Делеция** | DEL | Потеря участка ДНК | 50 bp – несколько Mb |
| **Дупликация** | DUP | Удвоение участка | 50 bp – несколько Mb |
| **Инверсия** | INV | Разворот участка | 50 bp – несколько Mb |
| **Инсерция** | INS | Вставка нового материала | 50 bp – несколько kb |
| **Транслокация** | BND | Перенос между хромосомами | Любая |
| **CNV** | CNV | Изменение числа копий | Любая |

## 3.4. Инструменты для детекции SV

### 3.4.1. На основе split-reads

| Инструмент | Подход | Преимущества |
|------------|--------|--------------|
| **Pindel** | Split-read analysis | Точные breakpoints |
| **Delly** | Combined approach | Хорошая чувствительность |
| **Lumpy** | Multi-signal integration | Объединяет несколько сигналов |

### 3.4.2. На основе discordant pairs

| Инструмент | Подход | Преимущества |
|------------|--------|--------------|
| **BreakDancer** | Pair-end mapping | Быстрый скрининг |
| **PEMer** | Statistical framework | Оценка значимости |

### 3.4.3. На основе coverage (для CNV)

| Инструмент | Подход | Преимущества |
|------------|--------|--------------|
| **CNVnator** | Read-depth analysis | Эффективен для крупных CNV |
| **Control-FREEC** | GC-normalized coverage | Учёт GC-bias |
| **GATK gCNV** | Bayesian model | Интеграция с GATK |

### 3.4.4. Комбинированные подходы

| Инструмент | Сигналы | Преимущества |
|------------|---------|--------------|
| **Manta** | Split + Discordant + Coverage | Быстрый, точный |
| **GRIDSS** | Assembly-based | Максимальная точность breakpoints |
| **SvABA** | Local assembly | Обнаружение сложных событий |

## 3.5. Аннотация SV

Аннотация структурных вариантов сложнее, чем для малых вариантов:

### Требования к аннотации

1. **Пересечение с генами/регуляторными областями** — какие гены затронуты?
2. **Оценка потери функции (loss of function)** — полный ли ген удалён?
3. **Популяционные частоты** — gnomAD-SV, DGV
4. **Предсказание эффекта на сплайсинг** — новые splice sites?
5. **Влияние на регуляторные элементы** — энхансеры, промоторы

### Инструменты аннотации

| Инструмент | Назначение |
|------------|------------|
| **VEP (с плагинами)** | Базовая аннотация SV |
| **SnpEff** | Ограниченная поддержка SV |
| **AnnotSV** | Специализированный аннотатор SV |
| **SVAnnotation** | Интеграция多个 источников |
| **gnomAD-SV browser** | Популяционные частоты SV |

### Важное ограничение

**VEP не покрывает SV так же хорошо, как малые варианты**, поэтому используют отдельные SV-аннотаторы.

## 3.6. Фильтрация и контроль качества SV

### Критерии качества

| Метрика | Порог | Описание |
|---------|-------|----------|
| **QUAL > 20** | Phred-качество | Минимальная уверенность |
| **SUPPORT ≥ 3** | Чтения поддержки | Минимум 3 чтения |
| **STRAND_BALANCE > 0.2** | Баланс цепей | Не менее 20% с каждой цепи |
| **MAPQ ≥ 30** | Качество картирования | Хорошее позиционирование |
| **GT = 0/1 или 1/1** | Генотип | Гетеро- или гомозигота |

### Красные флажки для SV 🚩

1. 🔴 Вариант в регионе с низкой mappability
2. 🔴 Поддержка только с одной цепи
3. 🔴 Breakpoint внутри повтора
4. 🔴 Низкое качество genotyping
5. 🔴 Отсутствие в базах данных (если не de novo)

## 3.7. Практический пайплайн детекции SV

```bash
#!/bin/bash
REF="hg38.fa"
BAM="sample.bam"
SAMPLE="S1"

# 1. Детекция SV через Manta
configManta.py --bam $BAM --referenceFasta $REF --runDir manta_run
manta_run/runWorkflow.py -m local -j 8

# 2. Конвертация и фильтрация
bcftools view manta_run/results/variants/diploidSV.vcf.gz | \
  bcftools filter -i 'QUAL>20 && INFO/SUPPORT>3' -O z -o $SAMPLE.sv.filt.vcf.gz

# 3. Аннотация через AnnotSV
annotSV -SVinputFile $SAMPLE.sv.filt.vcf.gz \
        -genomeBuild hg38 \
        -outputFile $SAMPLE.sv.annotated.tsv

# 4. Статистика
bcftools stats $SAMPLE.sv.filt.vcf.gz > $SAMPLE.sv.stats.txt
```

## 3.8. Визуализация SV

### IGV (Integrative Genomics Viewer)

```bash
# Загрузить BAM и VCF в IGV для ручной проверки
igv.sh -g hg38 sample.bam sample.sv.vcf
```

### Что проверять визуально:

1. ✅ Split reads на breakpoint
2. ✅ Discordant pairs правильной ориентации
3. ✅ Изменение покрытия (для CNV)
4. ✅ Отсутствие артефактов выравнивания
