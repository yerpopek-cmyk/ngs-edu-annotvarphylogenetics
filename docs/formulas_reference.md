# 📐 Сводка формул

Полный справочник всех формул, используемых в биоинформатическом анализе: аннотация генома, вызов вариантов, структурные варианты и филогенетика.

---

## 1. Формулы для аннотации генома

### 1.1. E-value (BLAST)

**Ожидаемое число случайных выравниваний с таким же или лучшим счётом:**

```
E-value = K × m × n × e^(-λ × S)
```

| Переменная | Описание | Типичные значения |
|------------|----------|------------------|
| `K` | Параметр Карлина-Альтшула (константа масштаба) | Зависит от матрицы замен |
| `m` | Длина запроса (query length) | 100–10 000 bp/aa |
| `n` | Общая длина базы данных | 10⁶–10¹¹ bp/aa |
| `λ` | Параметр скорости убывания (rate parameter) | ~0.267 для BLOSUM62 |
| `S` | Сырой счёт выравнивания (raw alignment score) | Зависит от матрицы |
| `e` | Основание натурального логарифма | 2.71828... |

**Типичный порог для уверенной гомологии:** `E-value ≤ 1e⁻⁵`

---

### 1.2. Связь E-value и Bit-score

```
E-value = m × n × 2^(-S')
```

| Переменная | Описание |
|------------|----------|
| `S'` | Bit-score (нормализованный счёт) |

**Преобразование raw score → bit-score:**

```
S' = (λ × S - ln K) / ln 2
```

---

### 1.3. N50 (оценка качества сборки)

**Алгоритм расчёта:**

```
1. Отсортировать контиги по убыванию длины: L₁ ≥ L₂ ≥ ... ≥ Lₙ
2. Вычислить общую длину: T = Σᵢ Lᵢ
3. Найти минимальное k, при котором: Σᵢ₌₁ᵏ Lᵢ ≥ 0.5 × T
4. N50 = Lₖ
```

**Пример:**

| Контиг | Длина (bp) | Кумулятивная сумма |
|--------|------------|-------------------|
| 1 | 5000 | 5000 |
| 2 | 3000 | 8000 |
| 3 | 2000 | 10000 ← N50 (достигнуто 50% от 20000) |
| 4 | 1500 | 11500 |
| 5 | 1000 | 12500 |

**Общая длина:** 12500 bp  
**50% от общей:** 6250 bp  
**N50:** 2000 bp (третий контиг)

## 2. Формулы для вызова вариантов

### 2.1. Phred-качество (QUAL)

```
QUAL = -10 × log₁₀(P(вариант ошибочный))
```

| QUAL | P(ошибки) | Точность | Интерпретация |
|------|-----------|----------|---------------|
| 10 | 0.1 | 90% | Низкое качество |
| 20 | 0.01 | 99% | Среднее качество |
| 30 | 0.001 | 99.9% | Хорошее качество ✅ |
| 40 | 0.0001 | 99.99% | Отличное качество |
| 50 | 0.00001 | 99.999% | Исключительное качество |

**Обратное преобразование:**

```
P(ошибки) = 10^(-QUAL / 10)
```

---

### 2.2. Genotype Likelihoods (PL)

```
PL = -10 × log₁₀(L)
```

| Переменная | Описание |
|------------|----------|
| `L` | Вероятность наблюдать данные при данном генотипе |

**Нормализация:** PL нормализованы так, что лучший генотип имеет `PL = 0`.

**Пример для диплоидного генотипа:**

| Генотип | L (вероятность) | PL (нормализованный) |
|---------|-----------------|---------------------|
| 0/0 | 0.00001 | 50 |
| 0/1 | 1.0 | 0 ← лучший |
| 1/1 | 0.01 | 20 |

---

### 2.3. Genotype Quality (GQ)

```
GQ = min(99, PL₂ - PL₁)
```

| Переменная | Описание |
|------------|----------|
| `PL₁` | PL лучшего генотипа (всегда 0) |
| `PL₂` | PL второго лучшего генотипа |

**Пример:**
- PL для 0/1 = 0 (лучший)
- PL для 0/0 = 30 (второй лучший)
- **GQ = 30 - 0 = 30**

---

### 2.4. Allele Balance (AB)

```
AB = ALT_count / DP
```

| Переменная | Описание | Ожидаемые значения |
|------------|----------|-------------------|
| `ALT_count` | Количество чтений с альтернативным аллелем | — |
| `DP` | Общая глубина покрытия | — |

**Интерпретация для диплоидного организма:**

| Генотип | Ожидаемый AB |
|---------|--------------|
| 0/0 (гомозигота реф) | ~0.0 |
| 0/1 (гетерозигота) | 0.4–0.6 |
| 1/1 (гомозигота альт) | ~1.0 |

---

### 2.5. Strand Bias (SB и FS)

**Упрощённый SB:**

```
SB = min(ALT_forward, ALT_reverse) / max(ALT_forward, ALT_reverse)
```

**GATK Fisher Strand (FS):**

```
FS = -10 × log₁₀(p)
```

где `p` — p-value теста Фишера на независимость распределения аллелей по цепям.

| FS | p-value | Интерпретация |
|----|---------|---------------|
| <10 | >0.1 | Нет strand bias ✅ |
| 10–30 | 0.001–0.1 | Слабый bias |
| 30–60 | 1e⁻⁶–0.001 | Умеренный bias ⚠️ |
| >60 | <1e⁻⁶ | Сильный bias 🔴 |

## 3. Формулы для филогенетики

### 3.1. Q-матрица (Neighbor-Joining)

```
Q(i, j) = (n - 2) × d(i, j) - Σₖ d(i, k) - Σₖ d(j, k)
```

| Переменная | Описание |
|------------|----------|
| `n` | Количество таксонов |
| `d(i, j)` | Расстояние между таксонами `i` и `j` |
| `Σₖ d(i, k)` | Сумма расстояний от `i` до всех остальных таксонов |
| `Σₖ d(j, k)` | Сумма расстояний от `j` до всех остальных таксонов |

**Пример расчёта для 4 таксонов:**

| | A | B | C | D | Σ |
|-|---|---|---|---|---|
| **A** | — | 5 | 7 | 10 | 22 |
| **B** | 5 | — | 8 | 11 | 24 |
| **C** | 7 | 8 | — | 6 | 21 |
| **D** | 10 | 11 | 6 | — | 27 |

**Q(A, B):**
```
Q(A, B) = (4 - 2) × 5 - 22 - 24
        = 2 × 5 - 46
        = 10 - 46
        = -36
```

---

### 3.2. Длины ветвей при объединении (NJ)

**Для таксонов f и g, объединяемых в узел u:**

```
d(f, u) = ½ d(f, g) + [1 / (2n - 2)] × (Σₖ d(f, k) - Σₖ d(g, k))
```

```
d(g, u) = d(f, g) - d(f, u)
```

**Пример продолжение (A и B объединяются):**

```
d(A, u) = ½ × 5 + [1 / (2×4 - 2)] × (22 - 24)
        = 2.5 + (1/6) × (-2)
        = 2.5 - 0.333
        = 2.167

d(B, u) = 5 - 2.167 = 2.833
```

---

### 3.3. Обновление матрицы расстояний (NJ)

**Расстояние от нового узла u до таксона x:**

```
d(u, x) = ½ [d(f, x) + d(g, x) - d(f, g)]
```

**Пример (u до C):**

```
d(u, C) = ½ [d(A, C) + d(B, C) - d(A, B)]
        = ½ [7 + 8 - 5]
        = ½ × 10
        = 5
```

---

### 3.4. Теорема Байеса (Bayesian phylogenetics)

```
P(B|A) = P(A|B) × P(B) / P(A)
```

**Для филогенетики:**

```
P(Tree|Data) = P(Data|Tree) × P(Tree) / P(Data)
```

| Член | Название | Описание |
|------|----------|----------|
| `P(Tree|Data)` | Апостериорная вероятность | Вероятность дерева при данных |
| `P(Data|Tree)` | Правдоподобие (likelihood) | Вероятность данных при дереве |
| `P(Tree)` | Априорная вероятность | Предварительная вероятность дерева |
| `P(Data)` | Маргинальная вероятность | Нормирующая константа |

---

### 3.5. Robinson-Foulds distance

**Расстояние между двумя деревьями:**

```
RF(T₁, T₂) = |C(T₁) \ C(T₂)| + |C(T₂) \ C(T₁)|
```

| Обозначение | Описание |
|-------------|----------|
| `C(T)` | Множество нетривиальных клад дерева T |
| `\` | Разность множеств |
| `|...|` | Мощность множества (количество элементов) |

**Нормализованное RF расстояние:**

```
RF_norm = RF / (2n - 6)
```

где `n` — количество листьев (для бифурцирующего дерева).

| RF_norm | Интерпретация |
|---------|---------------|
| 0 | Деревья идентичны |
| 0–0.25 | Очень похожи |
| 0.25–0.5 | Умеренно различаются |
| 0.5–0.75 | Сильно различаются |
| >0.75 | Практически разные |

## 4. Дополнительные формулы

### 4.1. Модифицированная идентичность (KEGG KAAS)

```
Идентичность = pident × min(1, (L_aln × 2) / (L_query + L_target))
```

| Переменная | Описание |
|------------|----------|
| `pident` | Процент идентичности выравнивания |
| `L_aln` | Длина выравнивания |
| `L_query` | Длина запроса |
| `L_target` | Длина мишени |

---

### 4.2. Ti/Tv ratio (переходы/трансверсии)

```
Ti/Tv = N(переходы) / N(трансверсии)
```

| Тип мутации | Изменения | Ожидаемое соотношение |
|-------------|-----------|----------------------|
| **Переходы (Ti)** | A↔G, C↔T | ~2.0–2.2 (человек) |
| **Трансверсии (Tv)** | A↔C, A↔T, G↔C, G↔T | — |

**Интерпретация:**
- **Ti/Tv ≈ 2.0–2.2** — ожидаемое для человека ✅
- **Ti/Tv < 2.0** — возможный артефакт или отбор ⚠️
- **Ti/Tv > 2.5** — подозрительно высокое 🔴

---

### 4.3. Coverage (глубина покрытия)

```
Coverage = (N_reads × L_read) / L_genome
```

| Переменная | Описание | Пример |
|------------|----------|--------|
| `N_reads` | Количество чтений | 10 000 000 |
| `L_read` | Длина чтения | 150 bp |
| `L_genome` | Длина генома | 3 × 10⁹ bp (человек) |

**Пример:**
```
Coverage = (10⁷ × 150) / (3 × 10⁹)
         = 1.5 × 10⁹ / 3 × 10⁹
         = 0.5×
```

| Метрика | Порог | Интерпретация |
|---------|-------|---------------|
| **E-value** | ≤1e⁻⁵ | Значимая гомология |
| **QUAL** | ≥30 | Хорошее качество варианта |
| **GQ** | ≥20 | Надёжный генотип |
| **DP** | ≥10 | Минимальная глубина |
| **MQ** | ≥30 | Хорошее картирование |
| **FS** | <60 | Приёмлемый strand bias |
