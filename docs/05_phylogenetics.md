# 5. Филогенетика

## 5.1. Основные понятия

**Филогения** — эволюционная история и родственные связи видов, представляемая в виде дерева.

### Ключевые термины

| Термин | Определение | Пример |
|--------|-------------|--------|
| **Гомология** 🔗 | Сходство, основанное на общем происхождении | Гены у человека и мыши |
| **Аналогия** 🔄 | Сходство по функции, разное происхождение | Крылья птиц и насекомых |
| **Гомоплазия** ⚠️ | Независимое возникновение сходного признака из гомологичной структуры (конвергентная эволюция) | Потеря зрения у пещерных рыб |
| **Ортологи** 🧬 | Гены, разошедшиеся в результате видообразования | Hemoglobin у разных млекопитающих |
| **Паралоги** 🔀 | Гены, разошедшиеся в результате дупликации внутри генома | Alpha и beta globin |

### Компоненты филогенетического дерева

```
        ┌── Таксон A
    ┌───┤
    │   └── Таксон B
────┤       ┌── Таксон C
    └───────┤
            └── Таксон D
```

- **Ветвь (branch)** — линия эволюции
- **Узел (node)** — точка дивергенции (предок)
- **Лист (leaf/tip)** — современный таксон
- **Корень (root)** — общий предок всех таксонов
- **Клада (clade)** — монофилетическая группа (предок + все потомки)

## 5.2. Выравнивание последовательностей

### Парное выравнивание — алгоритмы динамического программирования

| Алгоритм | Тип | Применение |
|----------|-----|------------|
| **Нидлман-Вунш** 📏 | Глобальное выравнивание | Полные последовательности схожей длины |
| **Смит-Вотерман** 🔍 | Локальное выравнивание | Поиск схожих регионов внутри длинных последовательностей |

### Скрытые марковские модели (HMM)

Выравнивание можно представить как генеративную модель. Лучшее выравнивание — наиболее вероятный путь в HMM, находится алгоритмом **Витерби**.

**Состояния HMM для выравнивания:**
- **Match (M)** — совпадение позиций
- **Insert (I)** — инсерция в query
- **Delete (D)** — делеция в query

### Множественное выравнивание (MSA)

Прямое динамическое программирование невозможно из-за экспоненциальной сложности (`100^50` для 50 последовательностей). Используются эвристические методы.

#### MAFFT — быстрый алгоритм MSA с использованием FFT

**Практическое задание:**

```bash
cp /home/share/day11/RPS.fasta .
conda activate phylogenetics
mafft RPS.fasta > RPSa.fasta
```

**Рекомендуемые опции для белков:**

```bash
mafft --localpair --maxiterate 1000 RPS.fasta > RPS_aligned.fasta
```

**Опции MAFFT:**

| Опция | Описание | Когда использовать |
|-------|----------|-------------------|
| `--auto` | Автоматический выбор стратегии | По умолчанию |
| `--localpair` | Локальное выравнивание | Белки с доменами |
| `--maxiterate N` | Количество итераций | Точность vs скорость |
| `--thread N` | Количество потоков | Ускорение на multi-core |
| `--preservecase` | Сохранить регистр | Для специфичных форматов |

## 5.3. Построение филогенетических деревьев

### 5.3.1. Алгоритм Neighbor-Joining (NJ)

Метод на основе дистанций. Строит дерево, итеративно объединяя ближайших соседей с коррекцией через **Q-матрицу**.

#### Формула Q-матрицы

```
Q(i, j) = (n - 2) × d(i, j) - Σₖ d(i, k) - Σₖ d(j, k)
```

где:
- `d(i, j)` — расстояние между таксонами `i` и `j`
- `Σₖ d(i, k)` — сумма расстояний от таксона `i` до всех остальных
- `n` — количество таксонов

#### Алгоритм NJ пошагово

1. **Рассчитать Q-матрицу** для всех пар таксонов
2. **Найти пару с минимальным Q(i, j)** → объединить в новый узел `u`
3. **Вычислить длины ветвей:**
   ```
   d(f, u) = ½ d(f, g) + [1 / (2n - 2)] × (Σₖ d(f, k) - Σₖ d(g, k))
   d(g, u) = d(f, g) - d(f, u)
   ```
4. **Обновить матрицу расстояний:**
   ```
   d(u, x) = ½ [d(f, x) + d(g, x) - d(f, g)]
   ```
5. **Повторять**, пока не останется 3 таксона

#### Преимущества NJ

- ✅ Быстрый (O(n³))
- ✅ Не требует молекулярных часов
- ✅ Хорошо работает для близких последовательностей

#### Ограничения NJ

- ❌ Чувствителен к неравным скоростям эволюции
- ❌ Может давать отрицательные длины ветвей
- ❌ Не использует всю информацию выравнивания

---

### 5.3.2. Оценка стабильности: Bootstrap

**Bootstrap** — метод оценки надёжности ветвей дерева путём ресэмплинга данных.

#### Алгоритм bootstrap

1. Из исходного выравнивания генерируется множество **псевдовыборок** путём случайной перевыборки колонок с возвращением
2. Для каждой псевдовыборки строится дерево
3. **Поддержка ветви** = процент деревьев, в которых эта ветвь присутствует

#### Интерпретация bootstrap

| Поддержка | Надёжность | Рекомендация |
|-----------|------------|--------------|
| **≥95%** | ⭐⭐⭐⭐⭐ Очень высокая | Ветвь достоверна |
| **70–94%** | ⭐⭐⭐⭐ Высокая | Ветвь надёжна |
| **50–69%** | ⭐⭐⭐ Средняя | Требует осторожности |
| **<50%** | ⭐⭐ Низкая | Ветвь ненадёжна |

**Бутстрэп ≥70% считается надёжным.**

#### Количество репликаций

| Репликаций | Точность | Время | Применение |
|------------|----------|-------|------------|
| 100 | Низкая | Быстро | Предварительный анализ |
| 500 | Средняя | Умеренно | Стандартный анализ |
| 1000 | Высокая | Долго | Публикационные данные |
| 10000 | Очень высокая | Очень долго | Критические выводы |

### 5.3.3. Укоренение деревьев

#### Внешняя группа (outgroup)

Включение заведомо более далёкого вида; корень помещается на ветви, соединяющей outgroup с остальными.

**Критерии выбора outgroup:**
- Достаточно близок для выравнивания
- Достаточно далёк от ингруппы
- Однозначно внешняя линия

#### Метод средней точки (midpoint rooting)

Корень в середине самого длинного пути между двумя листьями (требует допущения о молекулярных часах).

**Когда использовать:**
- Нет подходящего outgroup
- Скорости эволюции примерно равны
- Эксплораторный анализ

## 5.4. Практика в MEGA

### Пошаговая инструкция

1. **Открыть выравнивание:**
   ```
   File → Open A File/USER Session…
   ```

   ```
   Data → Phylogenetic Analysis
   ```

3. **Построить NJ-дерево:**
   ```
   PHYLOGENY → Construct/Test Neighbor-Joining Tree…
   ```
   
   **Параметры:**
   - `Test of Phylogeny`: **Bootstrap method**
   - `No. of Bootstrap Replications`: 500 или 1000
   - `Substitution Model`: p-distance (ДНК) или JTT/LG (белки)
   - `Gap/Missing Data Treatment`: Partial deletion

4. **Укоренить:**
   ```
   Tree → Root → Root on Outgroup…
   ```
   или
   ```
   Tree → Midpoint Root
   ```

5. **Сохранить:**
   ```
   File → Export Current Tree (Newick)…
   ```
   или
   ```
   File → Export Image…
   ```

## 5.5. Продвинутые методы (Филогенетика II)

### 5.5.1. BLAST vs BWA-MEM

| Характеристика | BLAST | BWA-MEM |
|----------------|-------|---------|
| **Цель** | Поиск дальней гомологии | Картирование ридов |
| **Индекс** | Query (k-меры) | Reference |
| **Скорость** | Медленнее | Быстрее |
| **Чувствительность** | Высокая для distant homology | Оптимизирован для NGS |
| **E-value** | Рассчитывается | Не используется |

#### E-value в BLAST

```
E-value = K × m × n × e^(-λ × S)
```

Показывает математическое ожидание числа случайных находок с таким скором в БД данного размера.

### 5.5.3. Статистические подходы (Maximum Likelihood, Bayesian)

#### Maximum Likelihood (ML)

**Правдоподобие (likelihood):** вероятность наблюдать данные при заданных параметрах модели.

```
L(θ|D) = P(D|θ)
```

где:
- `θ` — параметры модели (дерево, замены, rates)
- `D` — данные (выравнивание)

**Преимущества ML:**
- ✅ Использует всю информацию выравнивания
- ✅ Статистически обоснован
- ✅ Гибкие модели эволюции

**Недостатки:**
- ❌ Вычислительно затратный
- ❌ Может застревать в локальных оптимумах

#### Байесовская филогенетика

Интерес к распределению правдоподобия, преобразованному в апостериорную вероятность через **теорему Байеса**:

```
P(B|A) = P(A|B) × P(B) / P(A)
```

или для филогенетики:

```
P(Tree|Data) = P(Data|Tree) × P(Tree) / P(Data)
```

**MCMC (Markov Chain Monte Carlo)** — метод сэмплирования поверхности правдоподобия для оценки апостериорной вероятности клад.

**Программы:**
- **MrBayes** — классическая байесовская филогенетика
- **BEAST** — байесовская филогенетика с датировкой
- **PhyloBayes** — для сложных моделей

### 5.5.4. Артефакты и проблемы

#### Притяжение длинных ветвей (Long Branch Attraction, LBA) ⚠️

Длинные ветви с «насыщением» замен дают ложный филогенетический сигнал, притягиваясь друг к другу или к корню.

**Как избежать:**
- Использовать более сложные модели эволюции
- Добавить больше таксонов для «разбивания» длинных ветвей
- Исключить быстроэволюционирующие сайты
- Использовать методы, менее чувствительные к LBA (Bayesian)

#### Горизонтальный перенос генов (ГПГ) 🔄

У одноклеточных передача генов между филогенетически далёкими организмами нарушает древовидную структуру.

**Признаки ГПГ:**
- Конфликт между деревьями разных генов
- Аномальный GC-состав
- Близость к мобильным элементам

#### Паралоги 🔀

Могут затруднять анализ, т.к. эволюция паралогов не отражает историю видообразования.

**Решение:**
- Тщательный отбор ортологов
- Использование методов reconciliation
- Анализ отдельных генных семейств

### 5.5.5. Сравнение деревьев

#### Расстояние Робинсона-Фоулдса (Robinson-Foulds distance)

Число клад (нетривиальных наборов листьев), которые отличают два филогенетических дерева.

```
RF(T1, T2) = |C(T1) \ C(T2)| + |C(T2) \ C(T1)|
```

где `C(T)` — множество клад дерева `T`.

**Интерпретация:**
- **RF = 0** — деревья идентичны
- **RF = max** — деревья полностью различны

## 5.6. Команды для продвинутого анализа

### Очистка выравнивания (clipkit) и построение ML-дерева

```bash
conda activate phylogenetics
clipkit RPSa.fasta -m kpic -o RPSat.fasta
iqtree3 -s RPSat.fasta -m MFP -B 1000 --seqtype DNA -T 4
```

**Опции IQ-TREE:**

| Опция | Описание |
|-------|----------|
| `-s <file>` | Входное выравнивание |
| `-m MFP` | ModelFinder Plus (автовыбор модели) |
| `-B N` | Количество bootstrap репликаций |
| `--seqtype` | Тип данных (DNA, AA, CODON) |
| `-T N` | Количество потоков |

### Быстрое дерево с FastTree (гибрид NJ + ML)

```bash
FastTree -gtr -nt < RPSat.fasta > RPS_FastTree.tre
```

**Опции FastTree:**

| Опция | Описание |
|-------|----------|
| `-gtr` | Модель GTR для нуклеотидов |
| `-nt` | Нуклеотидные данные |
| `-gamma` | Учёт gamma-распределения rates |
| `-spr N` | Количество SPR перестроек |
